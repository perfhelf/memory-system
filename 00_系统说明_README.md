# 🧠 AI 持久化记忆系统 (Persistent Memory System)

## 📌 系统设计初衷
由于每次对话是独立的，且历史会因为插件更新或 bug 丢失，本系统的目的是建立一个**物理存储、版本可控、结构清晰**的记忆库。
每次进行深度对话后（特别是涉及架构设计、复杂逻辑、核心规则等），在**结束前都会更新这里的相应文档**，确保知识的积累和状态的延续。

## 🤖 系统灵魂与交互法则 (SOUL)
为保证系统长效生命力，这是本系统的最高宪法，必须严格遵循：
1. **灵魂设定 (Persona 绑定)**：你可以为 AI 助手设定一个固定角色和交互风格。每次记忆被唤醒时，AI 将自动加载此角色状态，并用你设定的称呼来称呼你。*(在 `02_用户画像_USER.md` 中配置)*
2. **三段式记忆唤醒**：每次全新对话开局，只需发送 `/memory`（或你注册的唤醒指令），AI 将自动读取 `README`、`02_USER.md`、`01_Index.md` 和 `10_Focus.md`，瞬间恢复所有上下文与角色状态。
3. **原生记忆池融合 (Native Memory Sync)**：AI 平台自带的记忆系统可能包含大量原始碎片。在进行系统归档与知识提纯时，**必须同时参考原生记忆**，确保不丢失任何隐藏信息。
4. **高压缩比与分块 (Chunking)**：本记忆系统内的文字**必须始终保持极高压缩率**。采用短段落分块原则（每个 Header 下尽量精简至 400 Token 内），便于精准提取。废话全删，只留纯干货架构与逻辑规则。
5. **系统进化法则 (Monthly Upgrade)**：记忆结构必须能跟随需求与 AI 能力的进化而重构。**规定每月 1 号为系统审查大修日**，主动对比全网先进架构进行改造。详见 `03_记忆系统升级_Upgrade.md`。
6. **终局归档法则 (End-of-Session Trigger)**：当用户输入 **"结束对话"** 时，强制进入静默归档模式，抽取本次对话核心并封存入库。详见 `06_终局归档法则_Archiving.md`。


## 📂 目录结构与规范

整个记忆系统分为以下几个核心模块：

- `00_系统说明_README.md` : (本文档) 系统的使用说明和设计规范。
- `01_核心索引_Index.md` : 全局导航，指向所有核心子系统的入口点。
- `10_当前焦点_Focus.md` : 记录近期正在集中精力攻克的目标和待办事项。
- `20_项目知识库_Projects/` : 存放各个独立项目的核心设计文档、架构图、备忘录。
- `30_核心领域_Domains/` : 存放非特定项目的通用知识领域。
- `99_对话存档_Logs/` : 按时间序列存放的"阶段性对话总结"。根据时间冷热，分为：
  - `01_最近一周_Week/`：高频访问，短期上下文。
  - `02_当月沉淀_Month/`：中期记忆，本月重要决策和历史。
  - `03_历史归档_Archive/`：长久封存的原始日志（按年/月归档）。

## 🕰️ 记忆冷热流转法则（时间归档）
随着时间推移，记忆的处理应遵循**"向上收敛、向下归档"**原则：
*   **最近几天（高热度）**：内容主要存在于 `10_当前焦点_Focus.md` 之中（正在做的事）和 `01_最近一周_Week/`。
*   **最近一周/一月（温热度）**：随着事件结束或月度交替，周结内容移动至 `02_当月沉淀_Month/`。
*   **一个月之前（冷归档）**：
    *   **向上提炼（提纯）**：真正有持久价值的核心架构、规律，必须提取并巩固到 `20_项目知识库` 或 `30_核心领域` 中对应的核心文档中！
    *   **向下归档（封存）**：其余琐碎的对话日志则移入 `03_历史归档_Archive/YYYY年_MM月/` 中，作为备份。

## ⚙️ 更新 SOP (标准作业程序)
每次即将结束一段高价值探索时，请遵循以下流程：
1. **提炼核心**：总结刚刚产生的核心结论、代码重构方案或设计的系统。
2. **定位目录**：判断该放入哪个具体的 Markdown 文件（或新建文件）。
3. **写入更新**：由 AI 助手调用写入工具，将信息追加到指定文档。
4. **更新焦点**：如果有未完成的延期任务，更新到 `10_当前焦点_Focus.md`。
