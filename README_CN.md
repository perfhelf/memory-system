# 🧠 AI 持久化记忆系统

[English](./README.md) | **中文**

一个基于 **Markdown 文件** 的、可版本控制的 AI 助手持久化记忆框架。适用于 Claude、GPT、Gemini 等缺乏内置长期记忆的 AI 助手。

## 🤔 为什么需要这个系统？

每次 AI 对话都从零开始。聊天记录会丢失，上下文窗口会溢出，机构知识会蒸发。

本系统通过在本地文件系统维护一个 **结构化的 Markdown 记忆库**，让 AI 在跨会话间也能读写和记忆一切。

**核心哲学**：只要 AI 能读文件，它就能记住一切。

## ✨ 核心功能

| 功能 | 说明 |
| :--- | :--- |
| **一键冷启动** | 一条 `/memory` 命令，几秒内加载全量上下文 |
| **冷热分级** | 自动记忆老化：焦点 → 周报 → 月报 → 归档 |
| **高压缩比** | 每个段落 ≤ 400 tokens，实现精准检索 |
| **自动写入** | AI 在对话中静默归档有价值信息，无需手动操作 |
| **终局归档** | 对话结束时强制知识蒸馏与封存 |
| **月度自进化** | 内置每月 1 号系统自审与架构升级协议 |
| **技能检索** | AI 编码前必须查阅的技能索引，杜绝闭门造车 |
| **密钥管理** | 集中式密钥注册表，AI 自动读取，永不向用户索要 |
| **项目规则** | 按项目集中管理编码规范与架构约定 |

## 🚀 快速开始

### 1. 克隆仓库
```bash
git clone https://github.com/perfhelf/memory-system.git
```

### 2. 定制你的画像
编辑 `02_用户画像_USER.md`，填入你的身份、兴趣和技术偏好。

### 3. 注册唤醒命令
将 `.agents/workflows/memory.md` 接入你的 AI 平台的 workflow / slash-command 系统，并将文件内的路径替换为你的本地克隆路径。

### 4. 启动！
在 AI 对话中输入 `/memory`，系统将冷启动并加载你的全量上下文。

### 5. 开始积累知识
随着你与 AI 的协作，系统会自动填充：
- `10_当前焦点_Focus.md` — 活跃任务
- `20_项目知识库_Projects/` — 项目专属知识
- `30_核心领域_Domains/` — 领域专业知识
- `99_对话存档_Logs/` — 对话摘要

## 🏗️ 设计原则

1. **物理存储 > 聊天记录** — 磁盘上的文件可以跨平台迁移，永不丢失
2. **高压缩** — 每个段落 ≤ 400 tokens，实现外科手术级精准提取
3. **冷热分级** — 最近 = Focus，老化 = Logs，永久 = Domains
4. **无感归档** — AI 静默写入，绝不打断对话流问"要不要保存"
5. **月度进化** — 每月 1 号系统自审，对标全网最新 AI 记忆架构

## 🤝 适配平台

本框架与具体 AI 平台无关，只要你的 AI 助手支持**读写本地文件**即可使用：

- **Google Gemini** (via Antigravity / AI Studio)
- **Anthropic Claude** (via Claude Desktop / MCP)
- **OpenAI GPT** (via Custom GPTs with file access)
- **其他支持文件操作的 AI Agent 框架**
